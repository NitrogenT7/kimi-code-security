# Web Business Profile 输出模板

本模板供 SKILL.md 引用。写入项目 `AGENTS.md` 时，章节可根据应用类型删减，不要为填满而保留无关章节。

```markdown
## Web Business Profile

### 业务本质
用 2-4 句话说明这个 Web 应用的核心价值。例如：
"这是一个 B2C 电商平台，用户浏览商品、下单、支付，商家在后台管理商品和订单。核心资产是订单、支付流水、用户地址和优惠券。"

### 用户核心旅程
描述最常见的 1-3 条端到端流程，从入口到完成目标：
1. 匿名用户浏览 → 注册/登录 → 下单 → 支付 → 查看订单
2. 商家登录 → 上架商品 → 管理库存 → 处理退款
3. 管理员登录 → 审核内容/订单 → 修改系统配置

### 核心数据对象与生命周期
列出 2-5 种核心实体，以及它们的流转路径：
- 用户：注册 → 认证 → 角色变更 → 注销
- 订单：创建 → 待支付 → 已支付 → 发货 → 完成/退款
- 优惠券：发放 → 领取 → 使用 → 过期/回收
- 内容/文章：草稿 → 审核 → 发布 → 下架

### 角色与信任边界
| 角色 | 认证要求 | 典型能力 | 信任区域 |
|------|---------|---------|---------|
| 匿名访客 | 无 | 浏览公开内容 | 外部不可信区 |
| 普通用户 | 注册登录 | 管理自己的数据 | 认证用户区 |
| 会员/VIP | 付费/积分 | 使用特权功能 | 认证用户区 |
| 商家/创作者 | 额外资质 | 管理自己的资源 | 业务角色区 |
| 管理员 | 后台认证 | 管理全站数据和用户 | 管理员区 |
| 第三方回调 | Webhook/签名 | 被动接收事件 | 外部不可信区 |

### 关键链路
选择 2-4 条从用户操作到远端服务的完整链路，标注数据形态转换点：
1. 用户下单：前端表单 → HTTP JSON → 订单服务 → 数据库 → 支付网关回调 → 状态更新
2. 文件上传：浏览器文件 → multipart/form-data → 上传服务 → 对象存储 → CDN 分发
3. 第三方登录：OAuth 授权 → code → token 交换 → 用户信息同步 → 本地 session 签发

### 能力依赖
- 认证：Session / JWT / OAuth / SSO / 短信/图形验证码
- 支付：微信支付、支付宝、Stripe、PayPal 等回调
- 存储：本地文件系统 / 对象存储 / CDN
- 推送：邮件、短信、站内信、Webhook
- 第三方 SDK：地图、统计、社交分享、风控

### 值得关注的业务线索
基于以上分析，从业务角度指出哪些技术线索可能蕴含攻击面。只陈述事实和推断，不写具体行动建议：
- 支付回调直接修改订单状态，需确认签名与状态机校验
- 订单详情接口包含用户 ID 参数，需确认是否校验归属
- 优惠券可在关单后退回，需确认生命周期一致性
- 管理员后台与前台共用同一域名/认证域，需确认权限隔离
```
