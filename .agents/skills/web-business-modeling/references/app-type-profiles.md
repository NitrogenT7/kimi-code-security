# 常见 Web 应用类型业务画像提示

用于在分析时快速聚焦。不必回答所有问题，只挑对当前应用有价值的。

## 电商 / 零售

**核心问题**
- 谁在购买？谁在售货？平台是否同时服务买家和卖家？
- 订单状态机有几步？支付、发货、退款、售后分别由谁触发？
- 优惠体系有几种？优惠券、积分、会员价、新人礼能否叠加？
- 价格/库存由谁维护？商家后台能否修改已下单商品？

**关键数据对象**
用户、商品 SKU、库存、订单、订单项、支付流水、优惠券、地址、购物车、退款单

**信任边界关注点**
- 买家 ↔ 卖家 ↔ 平台三方权限
- 商家后台与前台是否分离认证
- 支付回调是否直接驱动订单状态

## SaaS / OA / 协作平台

**核心问题**
- 租户/组织/个人三种维度如何隔离？
- 资源（文档、项目、表格）的可见性粒度是什么？私有/组织内/公开/链接分享？
- 成员角色有几种？管理员能否看到所有成员数据？
- 是否有邀请、分享、导出、API Key、Webhook 等跨边界能力？

**关键数据对象**
租户、组织、用户、角色、资源、分享链接、API Key、Webhook、操作日志

**信任边界关注点**
- 同租户内不同成员的数据隔离
- 公开分享链接是否暴露过多信息
- 第三方集成（OAuth、Webhook）的权限范围

## 金融 / 钱包 / 支付

**核心问题**
- 资金流有几类？充值、提现、转账、支付、退款？
- 余额/资产是否分账户？是否有冻结、可用、在途状态？
- 风控在哪些节点介入？限额、二次验证、人工审核？
- 是否支持代扣、代付、分账、批量转账？

**关键数据对象**
用户、账户、余额、交易流水、提现申请、银行卡、实名信息、风控事件、对账单

**信任边界关注点**
- 资金操作的状态机是否可逆
- 回调/通知是否可伪造或重放
- 低权限用户能否触发高权限资金动作

## 内容 / 社区 / 社交

**核心问题**
- 内容生产者、消费者、审核者分别是谁？
- 内容有哪些可见性？公开、关注可见、私密、付费？
- 互动（点赞、评论、转发、私信）是否产生新的数据对象？
- 是否有创作者后台、广告投放、内容付费？

**关键数据对象**
用户、内容、评论、点赞、关注关系、私信、收藏、标签、审核记录、收益

**信任边界关注点**
- 用户 A 能否读取用户 B 的私密内容
- 已删除/下架内容是否仍可访问
- 私信/关系链是否可被枚举

## 票务 / 预约 / 教育

**核心问题**
- 被预约的资源是什么？座位、课程、医生、时间槽？
- 库存/容量如何扣减？下单锁定还是支付后锁定？
- 取消/改签/退款的规则是什么？资金如何退回？
- 是否有核销/签到/验证环节？

**关键数据对象**
资源、排期/场次、库存、预约单、订单、凭证/二维码、核销记录、退款单

**信任边界关注点**
- 库存超卖或重复预约
- 凭证是否可被伪造或转售
- 改签/取消是否绕过限制

## API / 数据服务

**核心问题**
- API 面向谁？内部服务、合作伙伴、公开开发者？
- 认证方式是什么？API Key、OAuth、签名、IP 白名单？
- 是否有速率限制、配额、计费？
- 数据返回粒度是否因调用方而异？

**关键数据对象**
应用/客户端、API Key、用户、资源对象、配额、账单、日志

**信任边界关注点**
- API Key 的权限范围与归属校验
- 内部 API 是否可被外部访问
- 批量接口是否导致越权或资源耗尽
